# ❗기억하려고 적어놓은 거❗

---

### 캐릭터 생성 및 데이터 입력
> ‼️목표: 플레이어의 기본 정보를 콘솔 창으로 입력받고, 객체에 저장하여 상태창으로 출력
- ``Console.ReadLine()``: 유저의 콘솔 입력 문자열 받기
- ``Split(',')``: 쉼표를 기준으로 문자열을 분할하여 배열로 저장

---

### 입력 데이터 유효성 검사 (입력 검증)
> ‼️목표: 유저가 잘못된 형식이나 너무 낮은 능력치를 입력했을 때, 에러로 튕기지 않고 올바른 값을 넣을 때까지 계속 재입력을 유도
- ``while(true)``와 ``break``: 올바른 값이 들어올 때까지 무한 루프를 돌리고, 조건을 만족하면 루프를 탈출하는 흐름
- ``float.TryParse()``: 숫자가 아닌 문자(예: "abc")를 입력했을 때 예외(Crash)가 발생하지 않도록 안전하게 숫자로 변환하고 성공 여부를 반환
- ``Trim()``: 입력된 문자열 앞뒤의 불필요한 공백을 제거하여 변환 오류 방지

---

### 능력치 강화 메뉴 및 상태 관리
> ‼️목표: 게임을 본격적으로 시작하기 전, 제한된 수량의 소비 재화(포션, 쿠폰)를 사용하여 캐릭터의 스탯을 전략적으로 강화할 수 있는 인터페이스
- ``switch-case`` **문**: 유저가 선택한 메뉴 번호에 따라 실행할 코드를 깔끔하게 분기 처리
- **재화 차감 로직** **(``--``)**: 아이템 성공 적용 시 보유 수량을 1씩 감소시키고 조건문(``if``)으로 잔여 수량 검증
- **플래그 변수** (``bGameStart``): ``bool`` 타입 변수를 활용하여 반복문의 유지 및 종료 상태를 관리
- 게임 시작 제어: ``0번``을 누르면 강화 루프를 완전히 종료하고 ``Game Start!!!!`` 메시지 출력

---

### 객체지향 설계 도입 및 직업 선택
> ‼️목표: 객체지향 프로그래밍(OOP)의 핵심 문법 **생성자**와 **프로퍼티**를 사용하여 클래스 구조를 안전하게 개선하고, 게임 시작 후 캐릭터를 성장시킬 수 있는 전직(직업 선택) 시스템
- **생성자 (Constructor)**: 객체가 메모리에 생성되는 타이밍에 이름, HP, MP, 공격력, 방어력을 강제하여 필수 데이터가 누락된 채 캐릭터가 만들어지는 것을 방지
- **프로퍼티 (Property - Getter)**: 클래스 내부의 필드 데이터를 외부에 무분별하게 노출하지 않고 안전하게 캡슐화하면서, 외부에서는 값만 안전하게 읽어갈 수 있도록 제어(``{ get; set; }`` 및 ``{ get; }``)
- **조건문을 활용한 데이터 동적 매칭**: 유저가 선택한 번호(1~4)에 따라 직업 문자열을 동적으로 할당하고, 조건에 맞지 않는 번호 입력 시 루프를 재실행하는 유효성 검증

---

### 생성자를 통한 초기화
> 💡(Player(string name, float hp, float mp, float power, float defence))

> ✨캐릭터가 처음 생성될 때 기본 직업("없음")과 레벨(1)

```csharp
public Player(string name, float hp, float mp, float power, float defence)
{
    Name = name;
    Hp = hp;
    Mp = mp;
    Attack = power;
    Deffence = defence;
    Job = "없음"; 
    Level = 1;    
}
```

---

### 멤버 변수(job, level) 추가 및 Getter 프로퍼티 작성
> Player 클래스의 맨 윗부분에 클래스 외부에서 안전하게 데이터를 가져갈 수 있도록 Getter 프로퍼티 형태
>    > 💡 특히 Name은 외부에서 임의로 수정할 수 없게 get;만 열어두어 Getter로 만듬

```csharp
public string Name { get; }      // Getter 프로퍼티
public string Job { get; set; }   // 직업 멤버 추가
public int Level { get; set; }    // 레벨 멤버 추가
```
